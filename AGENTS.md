# Codex Rules for ai_os_system

## Репозиторий

- Текущий репозиторий: `ai_os_system`.
- Текущий рабочий код Conductor: `conductor/`.
- `docs/architecture/` — архитектура реализации.
- `docs/data/` — данные и ownership.
- `docs/services/conductor/` — документация Conductor.
- `docs/product/use_cases/` — use cases.
- `knowledge/` — устойчивые правила.
- `knowledge/agents/` — агенты и роли.

## Порядок разработки

1. Сначала use case.
2. Потом ожидаемый результат.
3. Потом сущности и данные.
4. Потом правила классификации и маршрутизации.
5. Потом сервисы.
6. Потом ошибки.
7. Потом агенты, только если они реально нужны.
8. Потом задача для Codex.

## Правила работы Codex

- Работай только с указанными файлами. Не сканируй весь репозиторий. Не меняй не связанные файлы. Не переписывай архитектуру без отдельного согласования.
- Не меняй код без прямого запроса.
- Не переноси `conductor/` в `apps/conductor/` без отдельной миграционной задачи.
- Не создавай пустые целевые папки.
- Не добавляй новых агентов без use case.
