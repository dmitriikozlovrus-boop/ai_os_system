# Стандарт токеноэкономичной разработки модулей через Codex

## Главный принцип

Codex не проектирует систему.
Codex реализует заранее принятые решения.

Вся бизнес-логика, архитектура и правила должны быть определены до запуска Codex.

---

# Жизненный цикл разработки модуля

## Этап 1. Проектирование модуля

**Инструмент:** ChatGPT

**Результат:** `Module Brief.md`

### Назначение

Определить границы ответственности модуля.

### Структура

- Название
- Назначение
- Вход
- Выход
- Входит в модуль
- Не входит в модуль
- Зависимости

---

## Этап 2. Формализация правил

**Инструмент:** ChatGPT

**Результат:** `Module Rules.md`

### Назначение

Зафиксировать бизнес-логику модуля.

### Структура

- Правила
- Ограничения
- Обработка ошибок
- Приоритеты

---

## Этап 3. Проектирование данных

**Инструмент:** ChatGPT

**Результат:** `Data Contract.md`

### Назначение

Зафиксировать структуру входных и выходных данных.

### Структура

- Input
- Output
- Пример входа
- Пример выхода

---

## Этап 4. Подготовка задачи для Codex

**Инструмент:** ChatGPT

**Результат:** `Codex Task.md`

### Назначение

Подготовить узкое техническое задание для Codex.

### Структура

- Задача
- Разрешено изменять
- Запрещено изменять
- Используемые документы
- Критерий готовности
- Тесты

---

## Этап 5. Реализация

**Инструмент:** Codex

### Вход

- Module Brief
- Module Rules
- Data Contract
- Codex Task

### Результат

Измененный код.

---

## Этап 6. Проверка

**Инструмент:** ChatGPT или Codex

**Результат:** `Review Notes.md`

### Структура

- Что изменено
- Что проверить вручную
- Возможные риски
- Следующие задачи

---

# Минимальный комплект документов

Для небольшого модуля:

1. Module Brief.md
2. Module Rules.md
3. Codex Task.md

---

# Полный комплект документов

Для важного сервиса:

1. Module Brief.md
2. Module Rules.md
3. Data Contract.md
4. Codex Task.md
5. Review Notes.md

---

# Правила экономии токенов

1. Не просить Codex изучать весь проект.
2. Одна задача = один запуск.
3. Сначала принять решение, потом писать код.
4. Всегда ограничивать список файлов.
5. Большие задачи разбивать на этапы.
6. Давать Codex только необходимый контекст.

---

# Золотое правило

ChatGPT принимает решения.
Codex реализует решения.

Чем лучше подготовлены Module Brief, Module Rules и Codex Task, тем меньше токенов расходуется и тем выше качество результата.
